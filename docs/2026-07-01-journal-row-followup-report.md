## 4. 데이터 파일 교체 — 맨 위로 + 누르면 즉시 완전 재계산 + 부분 교체 ✅

말씀하신 취지("새 행 등록과 똑같이 부수 산출값·클러스터 배정을 완전 재계산")대로, **등록 파이프라인을 그대로 재사용**해 행에 고정(pin)했습니다.

- **위치:** 팝업 **맨 위**에 "데이터 파일 교체" 섹션(주황 박스). "저장 후 클러스터 재계산" 별도 버튼은 **삭제** — 교체 버튼 하나로 교체와 재계산이 동시에 일어납니다.
- **부분 교체 지원:** 한 행에 연결된 파일이 여러 개(시계열 등)면 드롭다운에서 **교체할 파일 1개를 선택**해 그것만 교체.
- **동작(누르면):**
  1. 확인창 → 기존 파일을 **`.bak` 백업**(`battery_visual_outputs/row_replace_backups/<row>_<stamp>/`로 이동, 데이터 루트 밖이라 재스캔 안 됨).
  2. 새 파일 배치(등록과 동일한 이름규칙 `{행}_{sample}_{프로토콜}…`). WRD면 journal 행의 areal density로 `mass_g`까지 넣어 **summary CSV(mAh/g)** 재생성(+옵션 시 raw_timeseries).
  3. **지표 재계산**(Rs/Rct/ICE 등 `compute_metrics`) → `summary_metrics.csv` upsert, dataset plot 재생성.
  4. **매칭 override 재지정**(옛 파일 제거, 새 파일 → 같은 행) → **매칭/클러스터 리포트 재작성**(`write_*_match_outputs`) → 잘못된 클러스터에서 올바른 클러스터로 자동 재배치.
  5. 캐시 무효화 후 팝업/툴팁 자동 새로고침.
  - **실험정보 입력칸은 건드리지 않습니다**(요청대로 데이터와 별개).
- 백엔드: `POST /api/journal/row-replace-file`(multipart: row, kind, target, file, write_raw) → `experiment_import.replace_journal_row_file`.
- 검증(임시 복사본): 합성 WRD 등록→행2 매칭→교체 호출 시 ①새 파일 디스크 존재 ②옛 파일 백업+삭제 ③override가 새 파일로 재지정(행2 유지) ④매칭 리포트 재생성 ⑤실험정보 셀(Sample="cell A") 불변 — 모두 통과. 잘못된 target은 400.
- ⚠️ 주의: capacity 행 교체 시, 그 셀의 폴더(`..._cyc/<date>/`)에 있던 **기존 `.wrd`도 함께 백업·교체**합니다(폴더가 셀 단위라 안전). EIS는 매칭 파일 1개만 교체.

## 5. §3 — CSV 사이클 용량곡선 & summary vs raw_timeseries (설명 추가)

맞습니다. **WRD는 사이클축 곡선을 보려면 CSV로 변환이 필요**하고, capacity live viewer의 "Summary source overlay"가 읽는 파일은 **이름에 `capacity`가 들어간 요약 CSV**입니다 — 공식 `*_Capacity.csv`이거나, 우리가 자동 생성한 **`{stem}_capacity_summary.csv`** 둘 중 디스크에 있는 것. 둘 다 **사이클별 한 줄**(Cycle, 충·방전 용량, CE…)이라 cycle이 축이 됩니다.

- `{stem}_capacity_summary.csv` (**항상 생성**) = 사이클축 요약. 용량곡선·KPI·클러스터에 쓰는 그 파일.
- `{stem}_raw_timeseries.csv` (**체크박스 켤 때만**) = 원시 측정 레코드(시간·전압·전류·누적용량). cycle 축이 아니라 시간축 전압 프로파일/디버깅용, 용량이 큼.
- **'새실험등록' 위저드의 "WRD raw time-series CSV도 생성" 체크박스 옆에 위 설명을 추가**했습니다.

## 7. 새 실험 등록 append 위치 버그 수정 ✅ (511/512 오해 관련)

- **511/512는 빈 행이 아닙니다** — 실제 ZIB 실험(12파이_Ti foil, 2M ZnSO4, sample "pure OMC 712 3T_3_100/4_1000")입니다. 데이터 파일이 매칭 안 돼서 고아로 표시된 것이 맞습니다. (빈 행 510·513은 고아로 표시 안 됨 — 확인함.)
- **진짜 버그 발견:** 마지막 실데이터는 512행인데, openpyxl `max_row`가 **631**(513~631은 완전 빈 행인데 잔여 서식 때문에 카운트됨). 기존 `append_journal_row`가 `max_row+1`을 써서 **새 실험이 632행에 등록될 뻔했습니다**(510~631 대량 공백).
- **수정:** `last_populated_row()`(값이 실제로 있는 마지막 행 탐색)를 추가해 그 다음 행에 등록. 이제 **새 실험은 513행부터** 들어갑니다. (510 공백은 511/512가 다른 화학(ZIB)이라 생긴 구분 여백으로 보여 그대로 뒀습니다 — 굳이 채우길 원하면 알려주세요.)

## 8. 클러스터 없는 매칭 파일 → 1파일 클러스터 + 고아 파일 표 (3차)

**A. "파일은 있는데 클러스터만 없는 행" = EIS 9개(행 443~450, 506), Capacity 0개.**
- 이들은 매칭(행 연결)은 됐지만 비교 파트너가 없어 다중파일 비교 클러스터에 못 끼던 EIS 단일 측정들(대부분 `_2nd_`, `6hr again` 등)입니다. 기존엔 개별(source) 모드에서만 보였습니다.
- **수정:** 이런 확정매칭(verified/manual) 파일을 EIS 비교 드롭다운에 **자기 자신만의 1파일 클러스터**(`IND::<경로>`, 라벨 "EIS 단일 … 1 file")로 자동 노출. 오버레이도 그 1파일을 렌더하도록 처리. (매칭 코어는 건드리지 않고 뷰어/오버레이 계층에만 추가 — 회귀위험 낮음.) Capacity는 모든 파일이 이미 클러스터에 들어가 있어 대상 0.
- 검증: EIS 비교 드롭다운 12개 = 실클러스터 2 + 1파일 클러스터 9 + 전체 aggregate 1. IND 항목 렌더 정상(series 1).

**B. 고아 파일 섹션(실험일지 아래) = 139개(전부 EIS).**
- 어느 행에도 매칭 안 된 데이터 파일을 **파일명·경로·상태·생성일** 표로 실험일지 패널 밑에 표시(`GET /api/journal/orphan-files`). Capacity는 0(전부 매칭됨).
- "새로고침" 버튼 포함. 상태(unmatched/ambiguous/delete_candidate)도 같이 보여줘 왜 고아인지 파악 가능.
- 검증: 브라우저에서 139행 표 렌더 확인(첫 행: `new no3 GF C 2nd data_02.SEO`, 생성일 2026-04-23 14:04).

## 9. 무시행 표시 + 고아파일 레이아웃 마무리 (4차)

**A. 무시행(510/511/512) 숨김 + 빈 편집행 위치 교정.**
- 511/512(ZIB, 무시행 조건)는 '무시행 표시안함'에서 이미 source-row로는 숨겨졌지만, **빈 편집행이 510부터 깔리면서 511/512의 실제 데이터 위를 덮어 그리던 버그**가 있었습니다(잘못 덮어쓰면 데이터 손실 위험). 510(내부 빈 행)도 편집 여백처럼 보였고요.
- **수정:** `build_sheet_payload`의 빈 편집행 시작을 "마지막 표시행+1"이 아니라 **"마지막 데이터 존재행+1"**(513)로 변경(`_row_has_value` 추가). 결과: 무시행 표시안함 모드에서 510/511/512는 안 보이고, 새 데이터용 빈 편집행은 513부터 — append 위치(§7)와 정확히 일치.
- 기존 테스트 3개는 옛(덮어쓰던) 동작을 검증하고 있어 새 동작으로 갱신.
- 검증: sheet payload extraStartRow 510→513, 510/511/512 NOT SHOWN, 513+ 편집행.

**B. 고아파일 섹션 때문에 실험일지가 낑기던 문제.**
- 원인: 실험일지 페이지가 `height:100vh; overflow:hidden` + `journal-panel flex:1` 고정뷰포트 레이아웃이라, 고아파일 패널(형제 flex item)이 실험일지를 눌러 작아지게 만듦.
- **수정:** 고정뷰포트 잠금 해제 → 페이지 세로 스크롤 허용. 실험일지 뷰어는 `82vh`(min 520px)로 **메인 유지**, 고아파일 섹션은 그 아래로 깔려 **스크롤 내려서** 보이게.
- 검증: 실험일지 848px(뷰포트 82%)로 메인 유지, 고아파일은 y=1316(폴드 아래, 스크롤로 접근), 페이지 정상 스크롤.
